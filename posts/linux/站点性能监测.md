## 二、QPS与并发关系

```mermaid
flowchart LR
    A[并发数] --> B[响应时长]
    B --> C[QPS]
    
    C -->|低并发| D[QPS上升]
    C -->|峰值| E[最大QPS]
    C -->|过载| F[应用宕机]
    
    style E fill:#bfb,stroke:#333,stroke-width:2px
    style F fill:#f96,stroke:#333,stroke-width:2px
```

### QPS计算公式

```
QPS = 并发数 × (1000ms / 响应时长ms)
```

| 并发数 | 响应时长 | QPS |
|--------|----------|-----|
| 10 | 1ms | 10,000 |
| 100 | 500ms | 200 |
| 10000 | 崩溃 | N/A |

> 并发数增高 → 响应时间变长 → QPS先升后降 → 临界点为最大QPS

## 三、性能测试工具

### 3.1 hey 工具

HTTP负载生成器，用于Web应用性能测试。

```mermaid
flowchart TB
    A[hey命令] --> B[-n 总请求数]
    A --> C[-c 并发数]
    A --> D[-z 持续时间]
    A --> E[-o 输出格式]
    
    B --> B1[默认200]
    C --> C1[默认50]
    D --> D1[如: 10s, 3m]
    E --> E1[csv格式]
```

#### hey 使用示例

```bash
# 并发400 持续120秒
hey -z 120s -c 400 http://localhost:8080/school/list
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-n` | 总请求数 | 200 |
| `-c` | 并发数 | 50 |
| `-z` | 持续时间 | - |
| `-m` | HTTP方法 | GET |
| `-t` | 超时时间 | 20s |

#### hey 结果解析

```mermaid
flowchart TB
    A[hey输出] --> B[Summary统计]
    A --> C[Histogram响应时间分布]
    A --> D[Latency延迟分布]
    
    B --> B1[Total总时间]
    B --> B2[Requests/sec QPS]
    
    D --> D1[TP50/TP90/TP99]
    
    style B fill:#bfb,stroke:#333,stroke-width:2px
```

关键指标：
- **Requests/sec**: 吞吐量
- **Average**: 平均响应时间
- **TP99**: 99%请求的响应时间

### 3.2 wrk 工具

现代化HTTP基准测试工具，单CPU多核产生显著负载。

```mermaid
flowchart TB
    A[wrk命令] --> B[-c 连接数]
    A --> C[-t 线程数]
    A --> D[-d 持续时间]
    A --> E[--latency 延迟统计]
    
    B --> B1[TCP连接数<br/>=并发数]
    C --> C1[=CPU核数]
    D --> D1[压测时长]
    
    style A fill:#bbf,stroke:#333,stroke-width:2px
```

#### wrk 使用示例

```bash
# 10线程 400连接 持续120秒
wrk -c 400 -t 10 -d 120s -H "Authorization: Bearer pwd" --latency http://localhost:8080/user/list
```

### 6.2 MySQL连接池

```go
func InitDBClient(cfg *MysqlConfig) (*gorm.DB, error) {
    dsn := fmt.Sprintf("%s:%s@tcp(%s)/%s?charset=%s",
        cfg.User, cfg.Passwd, cfg.Addr, cfg.DB, cfg.Charset)

    db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{})
    if err != nil {
        return nil, err
    }

    sqlDB, _ := db.DB()
    sqlDB.SetMaxIdleConns(cfg.MaxIdleCount)
    sqlDB.SetMaxOpenConns(cfg.MaxOpenCount)

    return db, nil
}
```

### 6.3 Redis连接池

```go
func NewRedisAi() *RedisApi {
    pool := newRedisPoolWithSizeAndPasswd("127.0.0.1:6379", 1000, "")
    return &RedisApi{redisPool: pool}
}

func (api *RedisApi) Get(key string) (string, error) {
    redisConn := api.redisPool.Get()
    defer redisConn.Close()

    r, err := redisConn.Do("GET", key)
    return redis.String(r, err)
}
```

## 七、压测指标速查

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **QPS** | 每秒请求数 | 越高越好 |
| **TP50** | 50%请求响应时间 | < 50ms |
| **TP99** | 99%请求响应时间 | < 500ms |
| **Error Rate** | 错误率 | < 1% |
| **并发数** | 同时处理请求数 | 根据CPU核数 |
