# 算法

## 一、算法定义

算法是解决特定问题的步骤集合，是对问题求解方法的精确描述。

## 二、算法实例

**问题**：计算1+2+3+...+100的结果

### 1. 高斯算法
```python
sum = (1 + 100) * 100 / 2
```
**时间复杂度**：O(1)

### 2. 普通累加算法
```python
sum = 0
for i in range(1, 101):
    sum += i
```
**时间复杂度**：O(n)

## 三、算法时间复杂度

### 1. 定义
时间复杂度是衡量算法执行时间随输入规模增长的变化趋势，用大O符号表示：

```
T(n) = O(f(n))
```

- T(n)：算法的执行时间
- f(n)：输入规模n的函数
- O：表示渐近上界

### 2. 常见时间复杂度

| 复杂度 | 名称 | 描述 | 示例 |
|--------|------|------|------|
| O(1) | 常数阶 | 执行时间与输入规模无关 | 哈希表查找、数组访问 |
| O(log n) | 对数阶 | 执行时间随输入规模对数增长 | 二分查找 |
| O(n) | 线性阶 | 执行时间与输入规模线性增长 | 线性查找 |
| O(n log n) | 线性对数阶 | 执行时间与输入规模线性对数增长 | 归并排序、快速排序 |
| O(n²) | 平方阶 | 执行时间与输入规模平方增长 | 冒泡排序、选择排序 |
| O(n³) | 立方阶 | 执行时间与输入规模立方增长 | 矩阵乘法 |
| O(2ⁿ) | 指数阶 | 执行时间随输入规模指数增长 | 递归斐波那契 |
| O(n!) | 阶乘阶 | 执行时间随输入规模阶乘增长 | 旅行商问题暴力解法 |

### 3. 时间复杂度排序
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

## 四、空间复杂度

### 1. 定义
空间复杂度是衡量算法所需存储空间随输入规模增长的变化趋势：

```
S(n) = O(f(n))
```

### 2. 常见空间复杂度
- O(1)：常数空间，与输入规模无关
- O(n)：线性空间，与输入规模线性相关
- O(n²)：平方空间，与输入规模平方相关

## 五、算法设计原则

1. **正确性**：算法能够正确解决问题
2. **可读性**：算法代码清晰易懂
3. **效率**：时间和空间复杂度合理
4. **健壮性**：能够处理边界情况和异常输入

## 六、算法优化方向

1. **时间优化**：选择更高效的算法和数据结构
2. **空间优化**：减少内存使用，避免不必要的空间分配
3. **代码优化**：减少循环次数，避免重复计算
4. **并行优化**：利用多线程或分布式计算提高效率
