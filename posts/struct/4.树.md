# 树

## 一、树的定义

树是一种非线性数据结构，由n(n≥0)个节点组成，具有以下特征：
- 有一个特殊的节点称为根节点
- 其余节点分为若干个互不相交的集合，每个集合本身又是一棵树，称为子树

## 二、树的基本概念

| 术语 | 定义 |
|------|------|
| 根节点 | 树的最顶端节点，没有父节点 |
| 父节点 | 有子节点的节点 |
| 子节点 | 被父节点直接包含的节点 |
| 叶子节点 | 没有子节点的节点 |
| 节点的度 | 一个节点拥有的子节点数量 |
| 树的度 | 树中所有节点的最大度 |
| 深度 | 从根节点到该节点的路径长度 |
| 高度 | 从该节点到最远叶子节点的路径长度 |
| 路径 | 从一个节点到另一个节点的节点序列 |
| 路径长度 | 路径中节点的数量减1 |
| 节点的权 | 赋予节点的数值 |
| 带权路径长度 | 从根节点到该节点的路径长度与节点权的乘积 |

## 三、常见的树结构

### 1. 二叉树
- **定义**：每个节点最多有两个子节点的树
- **特点**：左子节点和右子节点有顺序之分

### 2. 满二叉树
- **定义**：除叶子节点外，所有节点都有两个子节点的二叉树
- **特点**：叶子节点都在同一层，节点总数为2ⁿ-1（n为层数）

### 3. 完全二叉树
- **定义**：除去最后一层为满二叉树，且最后一层的节点从左到右依次排列
- **判断方法**：按满二叉树的结构逐层顺序编号，无空档即为完全二叉树

### 4. 二叉查找树
- **定义**：左子树所有节点值小于根节点，右子树所有节点值大于根节点
- **特点**：查找效率高，平均时间复杂度为O(log n)
- **缺点**：可能退化为链表，最坏时间复杂度为O(n)

### 5. 平衡二叉树（AVL树）
- **定义**：二叉查找树且任意节点的左右子树高度差不超过1
- **特点**：通过旋转操作保持平衡，查找时间复杂度稳定为O(log n)

### 6. 红黑树
- **定义**：自平衡二叉查找树，节点分为红色和黑色
- **特点**：通过颜色规则保持平衡，插入删除操作时间复杂度为O(log n)
- **应用**：Java的TreeMap、HashMap（JDK 8+）

### 7. 霍夫曼树
- **定义**：带权路径长度最短的二叉树，也称为最优二叉树
- **应用**：数据压缩、编码

### 8. B树
- **定义**：平衡的多路搜索树
- **特点**：每个节点可以有多个子节点，节点存储数据
- **应用**：文件系统、数据库索引

### 9. B+树
- **定义**：B树的变体，数据只存储在叶子节点
- **特点**：叶子节点通过链表连接，支持范围查询
- **应用**：MySQL InnoDB存储引擎索引

## 四、霍夫曼树构建

**步骤**：
1. 将所有节点按权值从小到大排序
2. 取出权值最小的两个节点，创建一个新节点作为它们的父节点，父节点权值为两节点权值之和
3. 将新节点放回序列，重复步骤1-2直到只剩一个节点

**示例**：构建权值为A1、B2、C3、D4、E5的霍夫曼树
- 合并A1和B2 → F3
- 合并F3和C3 → G6
- 合并D4和E5 → H9
- 合并G6和H9 → I15

## 五、B树与B+树的区别

| 特性 | B树 | B+树 |
|------|------|------|
| 数据存储 | 所有节点都存储数据 | 只在叶子节点存储数据 |
| 叶子节点 | 无特殊连接 | 通过链表连接 |
| 查找过程 | 找到数据就返回 | 必须找到叶子节点 |
| 范围查询 | 效率低 | 效率高 |
| 扇出 | 较小 | 较大 |
| 高度 | 较高 | 较低 |

## 六、树的应用场景

| 树结构 | 应用场景 |
|--------|----------|
| 二叉查找树 | 有序数据的快速查找 |
| 红黑树 | 内存中的有序数据结构 |
| B+树 | 磁盘中的索引结构 |
| 霍夫曼树 | 数据压缩、编码 |
| 线段树 | 区间查询、范围更新 |
| Trie树 | 字符串前缀查询 |

## 七、面试常见问题

### 1. InnoDB一棵B+树可以存放多少行数据？
- 假设页大小为16KB，索引键为8字节，指针为6字节
- 非叶子节点：每个节点可存储约1170个索引键
- 叶子节点：每个节点可存储约150行数据
- 3层B+树可存储约1170×1170×150≈2000万行数据

### 2. 为什么索引结构默认使用B+树？
- **Hash**：适合等值查找，不适合范围查询
- **二叉树**：高度不均匀，查找效率不稳定
- **红黑树**：树高度较高，磁盘IO次数多
- **B树**：非叶子节点存储数据，扇出小，树高度高
- **B+树**：
  - 叶子节点链表连接，支持范围查询
  - 非叶子节点只存索引，扇出大，树高度低
  - 磁盘IO次数少，查询效率高
