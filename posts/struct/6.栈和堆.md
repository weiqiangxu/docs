# 栈和堆

## 一、内存角度的栈和堆

### 1. 栈内存
- **定义**：栈内存是由编译器自动分配和释放的内存空间
- **特点**：
  - 空间较小，存储局部变量和函数调用信息
  - 操作速度快，遵循后进先出原则
  - 自动管理，不需要手动释放
- **存储内容**：
  - 函数参数
  - 局部变量
  - 函数返回地址
  - 函数调用栈帧

### 2. 堆内存
- **定义**：堆内存是由程序员手动分配和释放的内存空间
- **特点**：
  - 空间较大，存储动态分配的对象
  - 操作速度较慢，需要手动管理
  - 可能产生内存泄漏
- **存储内容**：
  - 动态分配的对象
  - 数组
  - 大对象

### 3. 栈与堆的区别

| 特性 | 栈 | 堆 |
|------|------|------|
| 分配方式 | 自动分配 | 手动分配 |
| 释放方式 | 自动释放 | 手动释放 |
| 空间大小 | 较小 | 较大 |
| 访问速度 | 快 | 慢 |
| 内存碎片 | 无 | 可能产生 |
| 存储内容 | 局部变量、函数调用 | 动态对象、数组 |

## 二、数据结构角度的栈和堆

### 1. 栈（数据结构）
- **定义**：栈是一种只能从一端存取数据的线性结构，遵循后进先出原则
- **操作**：压栈（push）、弹栈（pop）、查看栈顶（peek）
- **应用**：函数调用、表达式求值、括号匹配

### 2. 堆（数据结构）
- **定义**：堆是一种特殊的完全二叉树，分为最大堆和最小堆
  - **最大堆**：每个节点的值都大于或等于其子节点的值
  - **最小堆**：每个节点的值都小于或等于其子节点的值
- **操作**：插入、删除、建堆
- **应用**：优先队列、堆排序、图算法

## 三、堆排序

### 1. 基本思想
- 利用堆这种数据结构所设计的一种排序算法
- 将待排序序列构造成一个最大堆
- 将堆顶元素与末尾元素交换，将最大元素"沉"到数组末端
- 重新调整结构，使其满足堆定义，然后继续交换堆顶元素与当前末尾元素，反复执行

### 2. 时间复杂度
- 建堆：O(n)
- 排序：O(n log n)
- 整体时间复杂度：O(n log n)

### 3. 空间复杂度
- O(1)，原地排序

### 4. 代码示例（Python）
```python
def heapify(arr, n, i):
    largest = i
    l = 2 * i + 1
    r = 2 * i + 2
    
    if l < n and arr[l] > arr[largest]:
        largest = l
    if r < n and arr[r] > arr[largest]:
        largest = r
    
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

def heap_sort(arr):
    n = len(arr)
    
    # 建堆
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    
    # 排序
    for i in range(n - 1, 0, -1):
        arr[i], arr[0] = arr[0], arr[i]
        heapify(arr, i, 0)
    
    return arr
```

## 四、优先队列

### 1. 定义
优先队列是一种特殊的队列，其中每个元素都有一个优先级，出队时按优先级顺序出队

### 2. 实现
- 通常使用堆实现
- 最大堆实现最大优先队列
- 最小堆实现最小优先队列

### 3. 应用
- 任务调度
- 事件处理
- Dijkstra算法
- Huffman编码

## 五、内存管理

### 1. 内存分配策略
- **栈分配**：编译器自动分配，速度快
- **堆分配**：通过malloc、new等函数手动分配

### 2. 内存泄漏
- **定义**：程序中已动态分配的堆内存由于某种原因未释放，导致内存浪费
- **避免方法**：
  - 及时释放动态分配的内存
  - 使用智能指针
  - 注意异常处理

### 3. 内存碎片
- **定义**：堆内存中存在的无法被有效利用的小块内存
- **减少方法**：
  - 使用内存池
  - 合理分配内存大小
  - 定期整理内存
