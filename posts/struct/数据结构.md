# 数据结构

## 一、数据结构概述

### 1. 数据结构起源
数据结构是一门研究非数值计算的程序设计问题中的操作对象，以及它们之间的关系和操作等相关问题的学科。

**核心公式**：程序设计 = 数据结构 + 算法

### 2. 逻辑结构
逻辑结构是指数据对象中数据元素之间的相互关系：
- **集合结构**：数据元素之间只有“同属于一个集合”的关系
- **线性结构**：数据元素之间存在一对一的线性关系
- **树形结构**：数据元素之间存在一对多的层次关系
- **图形结构**：数据元素之间存在多对多的任意关系

### 3. 物理结构
物理结构是指数据的存储结构，即数据元素在计算机中的存储方式：
- **顺序存储**：数据元素在内存中连续存储，如数组
- **链式存储**：数据元素在内存中不连续存储，通过指针链接，如链表

## 二、算法分析

### 1. 时间复杂度
时间复杂度是衡量算法执行时间随输入规模增长的变化趋势，用大O符号表示：

```
T(n) = O(f(n))
```

**常见时间复杂度**：
- O(1)：常数阶，如哈希表查找
- O(log n)：对数阶，如二分查找
- O(n)：线性阶，如线性查找
- O(n log n)：线性对数阶，如归并排序
- O(n²)：平方阶，如冒泡排序
- O(n³)：立方阶，如矩阵乘法
- O(2ⁿ)：指数阶，如递归斐波那契
- O(n!)：阶乘阶，如旅行商问题暴力解法

**时间复杂度排序**：
```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

### 2. 空间复杂度
空间复杂度是衡量算法所需存储空间随输入规模增长的变化趋势：

```
S(n) = O(f(n))
```

**常见空间复杂度**：
- O(1)：常数空间
- O(n)：线性空间
- O(n²)：平方空间

## 三、线性表

### 1. 定义
线性表是零个或多个数据元素的有限序列，具有以下特征：
- 数据元素之间存在一对一的线性关系
- 除第一个元素外，每个元素有且仅有一个直接前驱
- 除最后一个元素外，每个元素有且仅有一个直接后继

### 2. 顺序存储结构
- **定义**：用地址连续的存储单元存储线性表的数据元素，如数组
- **时间复杂度**：
  - 查找：O(1)
  - 插入：O(n)
  - 删除：O(n)
- **优点**：随机访问效率高，存储密度大
- **缺点**：插入删除操作效率低，需要预分配存储空间

### 3. 链式存储结构
- **定义**：数据元素在存储上不连续，每个元素包含数据域和指针域，如链表
- **时间复杂度**：
  - 查找：O(n)
  - 插入：O(1)（已知插入位置）
  - 删除：O(1)（已知删除位置）
- **优点**：插入删除操作方便，不需要预分配存储空间
- **缺点**：随机访问效率低，需要额外空间存储指针

### 4. 线性表分类
- **顺序表**：连续存储，随机访问快，插入删除慢
- **单链表**：链式存储，插入删除快，随机访问慢
- **双向链表**：双向遍历，操作方便
- **循环链表**：环形结构，遍历方便
- **静态链表**：用数组模拟链表，结合顺序存储和链式存储的优点

## 四、栈和队列

### 1. 栈
- **定义**：只能从一端（栈顶）存取数据的线性结构，遵循后进先出（LIFO）原则
- **操作**：压栈（push）、弹栈（pop）、查看栈顶（peek）
- **时间复杂度**：O(1)
- **应用**：函数调用、表达式求值、括号匹配、浏览器前进后退

### 2. 队列
- **定义**：只能从一端（队尾）插入数据，从另一端（队头）删除数据的线性结构，遵循先进先出（FIFO）原则
- **操作**：入队（enqueue）、出队（dequeue）、查看队头（front）
- **时间复杂度**：O(1)
- **应用**：任务调度、消息队列、缓冲区、广度优先搜索

### 3. 特殊队列
- **双端队列**：允许在两端进行插入和删除操作
- **优先队列**：元素具有优先级，出队时按优先级顺序出队
- **阻塞队列**：当队列为空时，出队操作会阻塞；当队列满时，入队操作会阻塞

## 五、树

### 1. 定义
树是一种非线性数据结构，由n(n≥0)个节点组成，具有以下特征：
- 有一个特殊的节点称为根节点
- 其余节点分为若干个互不相交的集合，每个集合本身又是一棵树，称为子树

### 2. 常见树结构
- **二叉树**：每个节点最多有两个子节点
- **满二叉树**：除叶子节点外，所有节点都有两个子节点
- **完全二叉树**：除去最后一层为满二叉树，且最后一层的节点从左到右依次排列
- **二叉查找树**：左子树所有节点值小于根节点，右子树所有节点值大于根节点
- **平衡二叉树**：二叉查找树且任意节点的左右子树高度差不超过1
- **红黑树**：自平衡二叉查找树，节点分为红色和黑色
- **B树**：平衡的多路搜索树
- **B+树**：B树的变体，数据只存储在叶子节点
- **霍夫曼树**：带权路径长度最短的二叉树

### 3. 树的应用
- **二叉查找树**：有序数据的快速查找
- **红黑树**：内存中的有序数据结构
- **B+树**：磁盘中的索引结构
- **霍夫曼树**：数据压缩、编码
- **堆**：优先队列、堆排序

## 六、图

### 1. 定义
图是一种非线性数据结构，用于表示对象之间的多对多关系。它由顶点（Vertex）集合和边（Edge）集合组成。

### 2. 图的类型
- **无向图**：边没有方向
- **有向图**：边有方向
- **完全图**：任意两个顶点之间都有边相连
- **加权图**：边有权重
- **有向无环图（DAG）**：有向且无环
- **二分图**：顶点分为两个集合，边只在集合间

### 3. 图的存储结构
- **邻接矩阵**：使用二维数组存储图的边信息
- **邻接表**：使用链表存储图的边信息

### 4. 图的遍历
- **深度优先搜索（DFS）**：从起始顶点出发，尽可能深地访问图中的顶点
- **广度优先搜索（BFS）**：从起始顶点出发，先访问其所有邻接顶点，然后再依次访问这些邻接顶点的邻接顶点

### 5. 图的算法
- **最短路径算法**：Dijkstra算法、Floyd-Warshall算法、Bellman-Ford算法
- **最小生成树**：Prim算法、Kruskal算法
- **拓扑排序**：对有向无环图的顶点进行排序
- **关键路径**：在带权有向无环图中，从源点到汇点的最长路径

## 九、总结

数据结构是计算机科学的基础，是程序设计的重要组成部分。掌握数据结构的基本概念和操作，对于提高程序设计能力和解决实际问题具有重要意义。

- **选择合适的数据结构**：根据问题的特点选择合适的数据结构，可以显著提高算法效率
- **理解算法复杂度**：分析算法的时间复杂度和空间复杂度，选择最优的算法实现
- **实践应用**：通过实际项目和编程练习，加深对数据结构的理解和应用能力

数据结构的学习是一个持续的过程，需要不断地学习和实践，才能真正掌握其精髓。
