# 图

## 一、图的定义

图是一种非线性数据结构，用于表示对象之间的多对多关系。它由顶点（Vertex）集合和边（Edge）集合组成。

## 二、图的基本概念

### 1. 顶点和边
- **顶点**：图中的节点，也称为节点
- **边**：连接两个顶点的线段

### 2. 有向图和无向图
- **无向图**：边没有方向，顶点A和顶点B之间的边可以双向遍历
- **有向图**：边有方向，顶点A到顶点B的边只能从A指向B
  - **弧尾**：有向边的起始点
  - **弧头**：有向边的终点

### 3. 完全图
- **无向完全图**：任意两个顶点之间都有边相连，有n个顶点的无向完全图有n(n-1)/2条边
- **有向完全图**：任意两个顶点之间都有两条方向相反的边，有n个顶点的有向完全图有n(n-1)条边

### 4. 度
- **无向图**：顶点的度是与该顶点相连的边的数量
- **有向图**：
  - 入度：指向该顶点的边的数量
  - 出度：从该顶点出发的边的数量

### 5. 路径和回路
- **路径**：从一个顶点到另一个顶点的顶点序列
- **路径长度**：路径中边的数量
- **回路**：起点和终点相同的路径
- **简单路径**：路径中没有重复顶点的路径
- **简单回路**：回路中除起点和终点外没有重复顶点的回路

## 三、图的存储结构

### 1. 邻接矩阵
- **定义**：使用二维数组存储图的边信息
- **优点**：查找边的时间复杂度为O(1)
- **缺点**：空间复杂度为O(n²)，适合稠密图

### 2. 邻接表
- **定义**：使用链表存储图的边信息
- **优点**：空间复杂度为O(n+e)，适合稀疏图
- **缺点**：查找边的时间复杂度为O(n)

## 四、图的遍历

### 1. 深度优先搜索（DFS）
- **基本思想**：从起始顶点出发，尽可能深地访问图中的顶点，直到无法继续前进时回溯
- **实现**：使用栈或递归
- **时间复杂度**：
  - 邻接矩阵：O(n²)
  - 邻接表：O(n+e)

### 2. 广度优先搜索（BFS）
- **基本思想**：从起始顶点出发，先访问其所有邻接顶点，然后再依次访问这些邻接顶点的邻接顶点
- **实现**：使用队列
- **时间复杂度**：
  - 邻接矩阵：O(n²)
  - 邻接表：O(n+e)

## 五、图的应用

### 1. 最短路径算法
- **Dijkstra算法**：用于计算从一个顶点到其他所有顶点的最短路径，适用于边权为非负的图
- **Floyd-Warshall算法**：用于计算所有顶点对之间的最短路径
- **Bellman-Ford算法**：用于计算从一个顶点到其他所有顶点的最短路径，适用于边权为负的图

### 2. 最小生成树
- **Prim算法**：从一个顶点开始，逐步添加边构建最小生成树
- **Kruskal算法**：按照边权从小到大添加边，避免形成环

### 3. 拓扑排序
- **定义**：对有向无环图（DAG）的顶点进行排序，使得对于每条有向边(u,v)，u在排序中都出现在v之前
- **应用**：任务调度、依赖分析

### 4. 关键路径
- **定义**：在带权有向无环图中，从源点到汇点的最长路径
- **应用**：项目管理、网络规划

## 六、代码示例

### 1. 邻接表实现图（Python）
```python
class Graph:
    def __init__(self):
        self.adj_list = {}
    
    def add_vertex(self, vertex):
        if vertex not in self.adj_list:
            self.adj_list[vertex] = []
    
    def add_edge(self, vertex1, vertex2, weight=None):
        self.add_vertex(vertex1)
        self.add_vertex(vertex2)
        self.adj_list[vertex1].append((vertex2, weight))
    
    def dfs(self, start_vertex):
        visited = set()
        result = []
        
        def dfs_helper(vertex):
            if vertex not in visited:
                visited.add(vertex)
                result.append(vertex)
                for neighbor, _ in self.adj_list[vertex]:
                    dfs_helper(neighbor)
        
        dfs_helper(start_vertex)
        return result
    
    def bfs(self, start_vertex):
        visited = set()
        result = []
        queue = [start_vertex]
        visited.add(start_vertex)
        
        while queue:
            vertex = queue.pop(0)
            result.append(vertex)
            for neighbor, _ in self.adj_list[vertex]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        
        return result
```

## 七、常见图算法

| 算法 | 用途 | 时间复杂度 |
|------|------|------------|
| Dijkstra | 单源最短路径 | O((n+e)log n) |
| Floyd-Warshall | 所有顶点对最短路径 | O(n³) |
| Prim | 最小生成树 | O((n+e)log n) |
| Kruskal | 最小生成树 | O(elog e) |
| 拓扑排序 | 有向无环图顶点排序 | O(n+e) |
| 深度优先搜索 | 图遍历 | O(n+e) |
| 广度优先搜索 | 图遍历 | O(n+e) |

## 八、图的实际应用

1. **社交网络**：用户作为顶点，好友关系作为边
2. **交通网络**：城市作为顶点，道路作为边
3. **计算机网络**：设备作为顶点，连接作为边
4. **知识图谱**：实体作为顶点，关系作为边
5. **推荐系统**：用户和物品作为顶点，交互作为边

## 九、图的类型

| 类型 | 特点 | 应用 |
|------|------|------|
| 无向图 | 边无方向 | 社交网络、交通网络 |
| 有向图 | 边有方向 | 网页链接、依赖关系 |
| 加权图 | 边有权重 | 交通网络、通信网络 |
| 有向无环图(DAG) | 有向且无环 | 任务调度、依赖分析 |
| 二分图 | 顶点分为两个集合，边只在集合间 | 匹配问题、推荐系统 |
