# 线性表

## 一、线性表定义

线性表是零个或多个数据元素的有限序列，具有以下特征：
- 数据元素之间存在一对一的线性关系
- 除第一个元素外，每个元素有且仅有一个直接前驱
- 除最后一个元素外，每个元素有且仅有一个直接后继

## 二、线性表实例

**生活实例**：幼儿园小朋友按顺序排队
- **优点**：查找某个小朋友位置容易
- **缺点**：插入或删除小朋友需要重新调整队伍

## 三、线性表顺序存储结构

### 1. 定义
顺序存储结构是用地址连续的存储单元存储线性表的数据元素，如数组。

### 2. 地址计算
假设线性表元素为 a₁, a₂, ..., aₙ，每个元素占据c个存储单元，则第i个元素的地址为：

```
Location(aᵢ) = Location(a₁) + c × (i-1)
```

### 3. 时间复杂度
- **查找**：O(1)，通过下标直接访问
- **插入**：O(n)，需要移动后续元素
- **删除**：O(n)，需要移动后续元素

### 4. 优缺点
- **优点**：随机访问效率高，存储密度大
- **缺点**：插入删除操作效率低，需要预分配存储空间

## 四、链式存储结构

### 1. 单链表
- **定义**：数据元素在存储上不连续，每个元素（节点）包含数据域和指针域，指针域指向下一个节点
- **时间复杂度**：
  - 查找：O(n)
  - 插入：O(1)（已知插入位置）
  - 删除：O(1)（已知删除位置）
- **优点**：插入删除操作方便，不需要预分配存储空间
- **缺点**：随机访问效率低，需要额外空间存储指针

### 2. 双向链表
- **定义**：每个节点有两个指针域，分别指向前驱和后继节点
- **优点**：可以双向遍历，方便查找前驱节点
- **缺点**：需要更多的存储空间

### 3. 循环链表
- **定义**：单链表的终端节点指针指向头节点，形成环形结构
- **优点**：从任意节点出发可以遍历整个链表

### 4. 静态链表
- **定义**：用数组模拟链表，每个元素包含数据域和游标（指向后继元素的数组下标）
- **优点**：结合了顺序存储和链式存储的优点

## 五、线性表分类

| 类型 | 存储方式 | 特点 | 适用场景 |
|------|----------|------|----------|
| 顺序表 | 连续存储 | 随机访问快，插入删除慢 | 元素数量固定，访问频繁 |
| 单链表 | 链式存储 | 插入删除快，随机访问慢 | 元素数量变化大，插入删除频繁 |
| 双向链表 | 链式存储 | 双向遍历，操作方便 | 需要频繁访问前驱和后继元素 |
| 循环链表 | 链式存储 | 环形结构，遍历方便 | 需要循环处理的场景 |
| 静态链表 | 数组存储 | 不需要指针，内存可控 | 无法使用指针的环境 |

## 六、线性表应用

1. **数组**：顺序存储的线性表，用于存储同类型数据
2. **链表**：链式存储的线性表，用于频繁插入删除的场景
3. **栈**：特殊的线性表，后进先出
4. **队列**：特殊的线性表，先进先出

## 七、术语解释

- **节点**：链表中的每个元素
- **数据域**：存储数据的部分
- **指针域**：存储下一个节点地址的部分
- **头节点**：链表的第一个节点
- **尾节点**：链表的最后一个节点
