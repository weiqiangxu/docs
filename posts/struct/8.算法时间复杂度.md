# 算法时间复杂度

## 一、时间复杂度定义

时间复杂度是衡量算法执行时间随输入规模增长的变化趋势，用大O符号表示：

```
T(n) = O(f(n))
```

- T(n)：算法的执行时间
- f(n)：输入规模n的函数
- O：表示渐近上界

## 二、常见时间复杂度

### 1. 常数阶 O(1)
- **定义**：执行时间与输入规模无关，始终为常数
- **示例**：
  - 哈希表查找
  - 数组访问
  - 基本算术运算
- **代码示例**：
```python
sum = (1 + 100) * 100 / 2  # 执行一次
print(sum)  # 执行一次
```

### 2. 对数阶 O(log n)
- **定义**：执行时间随输入规模对数增长
- **特点**：每次操作将问题规模减半
- **示例**：
  - 二分查找
  - 平衡二叉树查找
- **代码示例**：
```python
count = 1
while count < n:
    count *= 2  # 每次循环count翻倍
    # 时间复杂度为O(1)的操作
```

### 3. 线性阶 O(n)
- **定义**：执行时间与输入规模线性增长
- **示例**：
  - 线性查找
  - 遍历数组
- **代码示例**：
```python
for i in range(n):
    # 时间复杂度为O(1)的操作
```

### 4. 线性对数阶 O(n log n)
- **定义**：执行时间与输入规模线性对数增长
- **示例**：
  - 归并排序
  - 快速排序（平均情况）
  - 堆排序
- **特点**：将线性操作与对数操作结合

### 5. 平方阶 O(n²)
- **定义**：执行时间与输入规模平方增长
- **示例**：
  - 冒泡排序
  - 选择排序
  - 插入排序
- **代码示例**：
```python
for i in range(n):
    for j in range(n):
        # 时间复杂度为O(1)的操作
```

### 6. 立方阶 O(n³)
- **定义**：执行时间与输入规模立方增长
- **示例**：
  - 矩阵乘法
  - 三重循环算法

### 7. 指数阶 O(2ⁿ)
- **定义**：执行时间随输入规模指数增长
- **示例**：
  - 递归斐波那契
  - 暴力破解密码

### 8. 阶乘阶 O(n!)
- **定义**：执行时间随输入规模阶乘增长
- **示例**：
  - 旅行商问题暴力解法
  - 全排列生成

## 三、时间复杂度排序

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

## 四、时间复杂度分析方法

### 1. 大O表示法
- 只关注最高阶项
- 忽略常数系数
- 忽略低阶项

### 2. 最坏情况时间复杂度
- 算法在最坏情况下的执行时间
- 用于衡量算法的下界

### 3. 平均情况时间复杂度
- 算法在所有可能输入下的平均执行时间
- 更能反映算法的实际性能

### 4. 最好情况时间复杂度
- 算法在最好情况下的执行时间
- 参考价值有限

## 五、空间复杂度

### 1. 定义
空间复杂度是衡量算法所需存储空间随输入规模增长的变化趋势：

```
S(n) = O(f(n))
```

### 2. 常见空间复杂度
- O(1)：常数空间
- O(n)：线性空间
- O(n²)：平方空间

## 六、常见算法时间复杂度

| 算法 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 二分查找 | O(log n) | O(1) |
| 线性查找 | O(n) | O(1) |
| 归并排序 | O(n log n) | O(n) |
| 快速排序 | O(n log n) 平均 | O(log n) |
| 冒泡排序 | O(n²) | O(1) |
| 选择排序 | O(n²) | O(1) |
| 插入排序 | O(n²) | O(1) |
| 堆排序 | O(n log n) | O(1) |
| 计数排序 | O(n+k) | O(k) |
| 桶排序 | O(n+k) | O(n+k) |

## 七、时间复杂度优化技巧

1. **选择合适的算法**：根据问题特点选择时间复杂度更低的算法
2. **减少循环嵌套**：尽量减少多层循环，特别是内层循环的操作
3. **使用数据结构**：合理使用哈希表、二叉树等数据结构提高效率
4. **缓存计算结果**：避免重复计算，使用缓存存储中间结果
5. **并行计算**：利用多线程或分布式计算提高效率

## 八、实例分析

### 1. 斐波那契数列
- **递归实现**：O(2ⁿ)
- **动态规划**：O(n)
- **矩阵快速幂**：O(log n)

### 2. 查找算法
- **线性查找**：O(n)
- **二分查找**：O(log n)
- **哈希表查找**：O(1) 平均

### 3. 排序算法
- **冒泡排序**：O(n²)
- **快速排序**：O(n log n) 平均
- **归并排序**：O(n log n)

## 九、总结

- 时间复杂度是衡量算法效率的重要指标
- 选择合适的算法和数据结构可以显著提高程序性能
- 在实际开发中，需要根据具体问题选择时间复杂度和空间复杂度的平衡点
- 了解常见算法的时间复杂度有助于快速分析和优化代码
