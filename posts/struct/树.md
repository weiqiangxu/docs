# 树

## 一、树的定义

树是一种非线性数据结构，由n(n≥0)个节点组成，具有以下特征：
- 有一个特殊的节点称为根节点
- 其余节点分为若干个互不相交的集合，每个集合本身又是一棵树，称为子树

## 二、树的基本概念

| 术语 | 定义 |
|------|------|
| 根节点 | 树的最顶端节点，没有父节点 |
| 父节点 | 有子节点的节点 |
| 子节点 | 被父节点直接包含的节点 |
| 叶子节点 | 没有子节点的节点 |
| 节点的度 | 一个节点拥有的子节点数量 |
| 树的度 | 树中所有节点的最大度 |
| 深度 | 从根节点到该节点的路径长度 |
| 高度 | 从该节点到最远叶子节点的路径长度 |
| 阶（Order） | 节点的子节点数目的最大值 |
| 关键字 | 标识和区分数据记录的值 |
| 路径 | 从一个节点到另一个节点的节点序列 |
| 路径长度 | 路径中边的数量 |
| 节点的权 | 赋予节点的数值 |
| 带权路径长度 | 从根节点到该节点的路径长度与节点权的乘积 |

## 三、常见的树结构

### 1. 二叉树
- **定义**：每个节点最多有两个子节点的树
- **特点**：左子节点和右子节点有顺序之分
- **应用**：二叉查找树、平衡二叉树、红黑树

### 2. 满二叉树
- **定义**：除叶子节点外，所有节点都有两个子节点的二叉树
- **特点**：叶子节点都在同一层，节点总数为2ⁿ-1（n为层数）

### 3. 完全二叉树
- **定义**：除去最后一层为满二叉树，且最后一层的节点从左到右依次排列
- **判断方法**：按满二叉树的结构逐层顺序编号，无空档即为完全二叉树
- **应用**：堆排序、优先队列

### 4. 二叉查找树
- **定义**：左子树所有节点值小于根节点，右子树所有节点值大于根节点
- **特点**：查找效率高，平均时间复杂度为O(log n)
- **缺点**：可能退化为链表，最坏时间复杂度为O(n)

### 5. AVL平衡二叉树
- **定义**：二叉查找树且任意节点的左右子树高度差不超过1
- **特点**：通过旋转操作保持平衡，查找时间复杂度稳定为O(log n)
- **缺点**：插入删除操作需要多次旋转，时间复杂度较高

### 6. 红黑树
- **定义**：自平衡二叉查找树，节点分为红色和黑色
- **特点**：
  - 根节点是黑色
  - 红节点的子节点一定是黑色
  - 从根到叶子的所有路径都包含相同数目的黑色节点
- **时间复杂度**：查找、插入、删除均为O(log n)
- **应用**：Java的TreeMap、HashMap（JDK 8+）、Linux内核

### 7. 霍夫曼树
- **定义**：带权路径长度最短的二叉树，也称为最优二叉树
- **应用**：数据压缩、编码
- **构建方法**：
  1. 将所有节点按权值从小到大排序
  2. 取出权值最小的两个节点，创建一个新节点作为它们的父节点
  3. 将新节点放回序列，重复步骤1-2直到只剩一个节点

### 8. B树
- **定义**：平衡的多路搜索树
- **特点**：
  - 每个节点可以有多个子节点
  - 节点存储数据
  - 所有叶子节点在同一层
- **应用**：文件系统、数据库索引

### 9. B+树
- **定义**：B树的变体，数据只存储在叶子节点
- **特点**：
  - 非叶子节点只存储索引
  - 叶子节点通过双向链表连接
  - 支持范围查询
- **应用**：MySQL InnoDB存储引擎索引

## 四、B树与B+树的区别

| 特性 | B树 | B+树 |
|------|------|------|
| 数据存储 | 所有节点都存储数据 | 只在叶子节点存储数据 |
| 叶子节点 | 无特殊连接 | 通过双向链表连接 |
| 查找过程 | 找到数据就返回 | 必须找到叶子节点 |
| 范围查询 | 效率低 | 效率高 |
| 扇出 | 较小 | 较大 |
| 高度 | 较高 | 较低 |

## 五、树的应用场景

| 树结构 | 应用场景 |
|--------|----------|
| 二叉查找树 | 有序数据的快速查找 |
| 红黑树 | 内存中的有序数据结构 |
| B+树 | 磁盘中的索引结构 |
| 霍夫曼树 | 数据压缩、编码 |
| 线段树 | 区间查询、范围更新 |
| Trie树 | 字符串前缀查询 |
| 堆 | 优先队列、堆排序 |

## 六、常见问题解答

### 1. InnoDB一棵B+树可以存放多少行数据？
- 假设页大小为16KB，索引键为8字节，指针为6字节
- 非叶子节点：每个节点可存储约1170个索引键
- 叶子节点：每个节点可存储约150行数据
- 3层B+树可存储约1170×1170×150≈2000万行数据

### 2. 为什么索引结构默认使用B+树？
- **Hash**：适合等值查找，不适合范围查询
- **二叉树**：高度不均匀，查找效率不稳定
- **红黑树**：树高度较高，磁盘IO次数多
- **B树**：非叶子节点存储数据，扇出小，树高度高
- **B+树**：
  - 叶子节点链表连接，支持范围查询
  - 非叶子节点只存索引，扇出大，树高度低
  - 磁盘IO次数少，查询效率高

### 3. 如何选择合适的树结构？
- **内存数据**：红黑树、AVL树
- **磁盘数据**：B+树
- **有序数据**：二叉查找树
- **优先级队列**：堆
- **字符串处理**：Trie树
- **数据压缩**：霍夫曼树

## 七、树的遍历方法

### 1. 二叉树遍历
- **前序遍历**：根→左→右
- **中序遍历**：左→根→右
- **后序遍历**：左→右→根
- **层次遍历**：按层从左到右

### 2. 树的遍历应用
- **前序遍历**：打印树结构、复制树
- **中序遍历**：二叉查找树的有序遍历
- **后序遍历**：删除树、计算表达式
- **层次遍历**：广度优先搜索

## 八、总结

- 树是一种重要的非线性数据结构，广泛应用于计算机科学的各个领域
- 不同类型的树结构适用于不同的场景，选择合适的树结构可以显著提高算法效率
- 理解树的基本概念和操作是学习更复杂数据结构和算法的基础
- 树的应用非常广泛，从文件系统到数据库索引，从编译器到人工智能
